Meeting the Requirements of WCAG 2.1

  • Meeting level A and AA requirements are used by most regulatory standards (e.g. Section 508) as the mimimum for conformance.
  • None of the WCAG techniques are considered normative, but some are idenitfied as sufficent and others advisory techniques.
  • There are many accessibility techniques that are not part of the WCAG techniques document that can be used to meet success criteria requirements, so it is important to understand the accessibility issue being address by each success criteria.
  • Requirements like Success Criteria 2.1.1 Keyboard will have unique techniques for web applications based on the complexity of the user interaction and require an understanding of other accessibility specifications like ARIA to support consistent user experience or interoperabiliy.
